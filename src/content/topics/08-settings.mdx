- **ISMAP** が福祉施設の意思決定にどう関わるか

## なぜこのトピックが大事か

トピック⑦で「個人情報は AI に入れない」と書きました。とはいえ、業務上どうしてもグレーゾーンの情報を扱う場面はあります。そこで効くのが **「契約形態」** と **「設定」** の二段構え。

> **「設定でオフ」と「契約で対象外」は別物。** どちらも揃って初めて、施設の業務利用に耐えうる環境になります。

### Claude（個人アプリ / claude.ai）

[Anthropic の個人プラン（Free / Pro / Max）は **デフォルトで学習に使われません**](https://privacy.anthropic.com/en/articles/10023580-is-my-data-used-for-model-training)。アップロードしたファイルやチャット内容はモデル訓練に使用されない設定が標準で、明示的なオプトインが必要な仕組みです。

## 法人プランが担保する「学習オフ」以外の価値

| 価値 | 内容 |
|---|---|
| **データ学習対象外（既定）** | 設定ミスのリスクがそもそもない |
| **SSO / SCIM** | アカウント管理を施設の ID 基盤に統合できる |
| **監査ログ** | 「誰が、いつ、何を聞いたか」を法人側で記録できる |
| **管理者権限** | 退職者アカウントの即時無効化、利用範囲の制限 |
| **データ保持期間の制御** | 0 日保持 / 30 日保持 等を選べる場合がある |
| **DPA（データ処理契約）** | 個人情報保護法上の委託契約として根拠を持てる |
| **ISMAP 登録**（一部サービス） | 政府・自治体の調達基準を満たす |

## ISMAP とは（福祉施設の文脈で）

[**ISMAP**](https://www.ismap.go.jp/)（Information system Security Management and Assessment Program）= 政府情報システムのためのセキュリティ評価制度。**デジタル庁・総務省・経済産業省・国家サイバー統括室**（旧 NISC、2024 年に改称）が所管し、[**IPA（情報処理推進機構）が ISMAP 運用支援機関**](https://www.ipa.go.jp/security/ismap/index.html) として実務支援を担当します（令和 2 年 6 月開始）。

**福祉施設にとっての意味**：
- 「政府が事前にセキュリティ評価したリスト」に載っているサービスは、**説明責任を果たしやすい**
- 自治体との連携業務、行政との情報やり取りを含む場面で、**選定根拠として強い**
- 民間社会福祉法人でも、**「ISMAP 登録済を優先」** という内規を作ると、判断が早くなる

<Caution title="個人アカウント業務利用 = 事実上 NG" level="warn">
- 個人アカウントの ChatGPT/Gemini を業務利用すると、以下のリスクが発生します：
  - データ学習オフ設定漏れ
  - **退職者の業務情報持ち出し** が止められない
  - 監査ログがなく、事故時の追跡不可
  - 個人情報保護法上の **委託契約根拠が立たない**
- 「とりあえず個人アカウントで使ってる」は、**早期に法人契約に切り替える** ことを強く推奨します
</Caution>

<Caution title="「学習オフ ≠ サーバー送信なし」" level="info">
- 学習に使われなくても、入力データは AI ベンダーのサーバーに送信されます
- 保持期間・暗号化・アクセス制御は **各サービスの規約とプライバシーポリシーで確認**
- 完全ローカル処理（端末内のみ）を要する場合は、iPhone 純正「メモ」の文字起こしなどに選択肢を絞ることになります
</Caution>

## このトピックのまとめ

- **「設定で学習オフ」と「法人契約で対象外」は別物、両方そろえる**
- 個人プランでも学習オフは可能だが、**業務利用は法人契約を強く推奨**
- 法人契約の価値は学習対象外だけでなく、**SSO・監査ログ・DPA・管理者権限**
- 行政連携・要配慮情報を扱うなら **ISMAP 登録済サービス** を優先
- 仕様は半年単位で変わる、**契約更新時に再評価**
